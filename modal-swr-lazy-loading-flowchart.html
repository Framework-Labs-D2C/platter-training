Section 3: Product Data — SWR Caching (IndexedDB)

How individual product data is cached and revalidated across page loads

Entry — getHydratedProductData(handle)

Three-Tier Cascade

Called whenever a card or modal needs full product data. Tries three sources in order, returning the fastest available data while revalidating in background.

product-data.ts → lines 196–211
Tier 1: In-Memory
getHtmlProduct(handle)

Check window._products + inline JSON

Synchronous. Checks the in-memory product map, then looks for <script data-product-data> tags in the DOM. Fastest possible — no async.

product-data.ts → lines 10–44
Tier 2: IndexedDB
getCachedProduct(handle)

Check IndexedDB via idb-keyval

Async. Key: _{themeId}-product--{handle}. Returns cached data immediately. If _updated_at is >15min old, triggers a low-priority background fetch to revalidate.

product-data.ts → lines 45–72
Tier 3: Network
getFetchProduct(handle)

Fetch from Shopify Recommendations API

/recommendations/products?product_id=...§ion_id=data_product_json. Returns product + related products. Stores all to IndexedDB via requestIdleCallback.

Persist — saveProduct(handle)

Debounced IndexedDB Write

Uses requestIdleCallback with sequence tracking to handle rapid updates. Loops until all pending mutations are persisted. Prevents duplicate idbKeyval.set() calls.
