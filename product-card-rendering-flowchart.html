Static (Manual + Config)

  • Full HTML at request time — zero JS dependency
  • Fastest first contentful paint
  • SEO-friendly (crawlable product markup)
  • No layout shift (CLS = 0 for cards)
  • Works with JS disabled
  • Bracket notation resolved server-side

Lazy Hydration (Manual + Blank)

  • Product data IS server-embedded (inline JSON)
  • Only the card template is missing — no API fetch needed
  • Triggers on intersection (lazy) — defers JS cost
  • Merges into the same hydrateProductCard as dynamic
  • Common when metaobject not yet configured
  • Faster than full dynamic — skips data resolution

Full Dynamic (Recently Viewed)

  • No server HTML or data at all — fully client-driven
  • Personalized per-visitor (localStorage)
  • Targeting algorithms (AI, price sort, etc.)
  • Products fetched from IndexedDB or API
  • Card nodes cloned from template, then hydrated
  • Highest JS cost, most layout shift

Key Insight

  • The same card metaobject powers all three paths — the merchant designs the card once and the theme picks the render strategy.
  • Static = Shopify knows products + card config at request time
  • Lazy = Shopify knows products but card config is blank → merges into shared hydrateProductCard
  • Dynamic = Only the browser knows the products → same hydrateProductCard after data resolution

How data-style-id Differs: Static vs Dynamic

Same compiled CSS, different selector assignment strategies

● Static Cards

data-style-id="block_id" // Example: data-style-id="abc123" // CSS selector compiled as: [data-style-id="abc123"] { color: red; }

Set in Liquid by _render_blocks.liquid (line 7). The style_id equals the block_id from the metaobject's block_order. Every static card using the same metaobject shares identical block IDs, so they all match the same CSS rules.

card_product_card._render_blocks.liquid → line 7

● Dynamic Cards

data-style-id="section_id--block_id" // Example: data-style-id="sec_xyz--blk_789" // Set on wrapper div by JS: div.setAttribute("data-style-id", `${section_id}--${block_id}`);

Set in JavaScript by product-data.ts (line 1149). The compound format section_id--block_id is set on the wrapper <div> that contains the cloned card. The inner card nodes inherit block-level styles from the cloned template's own data-style-id attributes.
